Introducción a Git

¿Qué es Git y por qué es útil para el control de versiones?

Git es un sistema de control de versiones distribuido, creado por Linus Torvalds en 2005. Es una herramienta esencial para el desarrollo de software moderno, que permite a los desarrolladores rastrear cambios en el código, colaborar eficientemente y gestionar diferentes versiones de un proyecto.

Git destaca por varias razones:

  1. Historial completo: Git mantiene un registro detallado de todos los cambios realizados en un proyecto, permitiendo ver quién hizo qué cambios y cuándo.
  2. Trabajo distribuido: Cada desarrollador tiene una copia completa del repositorio, lo que facilita el trabajo sin conexión y mejora la redundancia.
  3. Branching y merging: Git permite crear ramas para trabajar en diferentes características o versiones del proyecto de forma aislada, y luego fusionarlas fácilmente.
  4. Velocidad y eficiencia: Git es notablemente rápido en comparación con otros sistemas de control de versiones, especialmente para operaciones locales.
  5. Flexibilidad de flujo de trabajo: Git se adapta a diversos estilos de desarrollo y tamaños de equipo.

Instalación de Git

La instalación de Git varía según el sistema operativo:

Windows

  1. Visita la página oficial de Git (https://git-scm.com/download/win) y descarga el instalador.
  2. Ejecuta el archivo descargado y sigue las instrucciones del asistente de instalación.
  3. Durante la instalación, asegúrate de seleccionar la opción para añadir Git al PATH del sistema.
  4. En la pantalla "Select Components", asegúrate de que la opción "Git Bash Here" esté marcada. Esto instalará Git Bash, que es el terminal que queremos usar.
  5. En la pantalla "Adjusting your PATH environment", selecciona la opción "Use Git from Git Bash only".
  6. Cuando llegues a la pantalla "Configuring the line ending conversions", es crucial que selecciones la siguiente opción: 
    Checkout Windows-style, commit Unix-style line endings
  7. Deja las demás opciones por defecto. instalación.

macOS

  1. La forma más sencilla es usar Homebrew. Si no lo tienes instalado, visita brew.sh para instrucciones.
  2. Abre la Terminal y ejecuta: brew install git.
  3. Alternativamente, puedes descargar el instalador desde la página oficial de Git: https://git-scm.com/downloads/mac
  4. Verifica la instalación con git --version en la Terminal.

Linux

  1. La mayoría de las distribuciones Linux incluyen Git preinstalado. Para verificar, abre una terminal y ejecuta git --version.
  2. Si no está instalado:
    • En distribuciones basadas en Debian/Ubuntu: sudo apt-get install git-all
    • En sistemas basados en Fedora: sudo dnf install git-all
    • En sistemas basados en ArchLinux: sudo pacman -S git
    • Para otras distribuciones, consulta la documentación específica o usa el gestor de paquetes correspondiente.

Después de la instalación, es recomendable configurar tu nombre de usuario y correo electrónico en Git usando los comandos:

        
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

Esta configuración es importante para identificar tus contribuciones en los proyectos.

Clonar el repositorio usando HTTPS y token

  1. Generar un token de acceso personal (PAT)
    1. En GitHub, haz clic en tu foto de perfil (esquina superior derecha) > Settings.
    2. En el menú izquierdo, ve a Developer settings > Personal access tokens > Tokens (classic).
    3. Haz clic en Generate new token > Generate new token (classic).
    4. Configura el token:
      • Note: Acceso a repositorios desde Git (descripción útil).
      • Expiration: Elige una fecha (ej: 30 días para el curso).
      • Scopes: Marca repo (para acceso completo a repositorios).
    5. Haz clic en Generate token.
    6. ¡Copia el token inmediatamente! No podrás verlo nuevamente.
    7. En el terminal ejecuta el siguiente comando: 
      git config --global credential.helper store
    8. Opcional: Crea o edita el archivo ~/.git-credentials (en Linux/macOS) o \Users\YourUsername\.git-credentials (en Windows) y añade una línea con este formato: 
      https://{username}:{personal_access_token}@github.com
      Reemplaza {username} con tu nombre de usuario de GitHub y {personal_access_token} con el token que generaste.
    9. Ve al repositorio: Perfil > your repositories > 2025Apellido
    10. En la página del repositorio, haz clic en Code > HTTPS y copia el enlace (ej: https://github.com/usuario/repo.git).
    11. Abre Git Bash/Terminal en el directorio donde lo quieras clonar y ejecuta:
      12. git clone https://github.com/usuario/repo.git
    12. Cuando te pida Username: Escribe tu nombre de usuario de GitHub.
    13. Cuando te pida Password: Pega el token generado (no verás caracteres al pegarlo).
    14. Pulsa Enter. El repositorio se clonará correctamente.

Comandos básicos de Git en el contexto de Overleaf

Nota importante: Si usas Overleaf con licencia Campus, Git está integrado y no necesitas usar la terminal. Sin embargo, si usas la versión gratuita o quieres trabajar localmente, estos comandos te serán útiles.

Contexto: ¿Cuándo usas Git desde terminal?

Si descargas tu proyecto desde Overleaf y quieres subirlo a GitHub (u otro repositorio), usarás estos comandos en tu terminal local:

git init

El comando git init se utiliza para inicializar un nuevo repositorio Git en el directorio actual. Esto crea un subdirectorio oculto .git que contiene toda la estructura necesaria para el control de versiones.

Ejemplo: Si descargas un proyecto desde Overleaf como ZIP, descomprímelo y ejecuta:

cd /ruta/a/tu/proyecto
git init

git status

git status muestra el estado actual del repositorio, incluyendo los archivos modificados, añadidos o eliminados que aún no se han confirmado.

git status

git add

El comando git add se usa para agregar archivos al área de preparación (staging area). Por ejemplo: 

git add <archivo>

Para añadir todos los archivos nuevos y modificados, puedes usar:

git add .

Para añadir todos los archivos nuevos y modificados y eliminar los borrados, puedes usar:

git add -A o git add --all

git commit

git commit crea una instantánea de los cambios en el área de preparación y la guarda en el historial del repositorio. Es recomendable incluir un mensaje descriptivo:

git commit -a -m "Descripción breve de los cambios realizados"

La opción -a en el comando git commit -a -m "mensaje" le indica a Git que incluya automáticamente todos los archivos modificados y eliminados en el commit, sin necesidad de usar git add para cada archivo. Sin embargo, no incluye nuevos archivos que no hayan sido previamente rastreados por Git.

Creación del repositorio remoto en GitHub

Antes de subir tu proyecto a GitHub, debes crear un nuevo repositorio remoto:

Crear un nuevo repositorio en GitHub:
  1. Ve a GitHub y haz clic en el botón New repository en la esquina superior derecha.
  2. Ingresa un nombre para tu repositorio y una descripción opcional.
  3. Selecciona la visibilidad del repositorio (público, privado o interno).
  4. Haz clic en Create repository sin inicializar el repositorio con un README, licencia o archivo .gitignore, ya que puedes agregar estos archivos después de subir tu proyecto.
Conectar el repositorio local con el remoto:
  1. En el repositorio de GitHub, copia la URL del repositorio que aparece en la página de configuración rápida.
  2. En tu terminal, navega al directorio de tu proyecto local.
  3. Agrega el repositorio remoto usando el comando: 
    git remote add origin https://github.com/tu-usuario/tu-repositorio.git
  4. Verifica que el remoto se haya agregado correctamente con: 
    git remote -v

git push

El comando git push se utiliza para enviar los commits locales a un repositorio remoto (GitHub). La sintaxis básica es:

git push <remoto> <rama>

Ejemplo típico:

git push origin main

¿Cuándo lo usas en Overleaf? Si descargas tu proyecto desde Overleaf, lo modificas localmente y quieres subirlo a GitHub:

git add .
git commit -m "Actualización desde local después de descargar de Overleaf"
git push origin main

git pull

git pull se usa para descargar y fusionar los cambios desde un repositorio remoto a tu repositorio local. Es una combinación de git fetch y git merge.

Ejemplo:

git pull origin main

¿Cuándo lo usas en Overleaf? Si un compañero subió cambios a GitHub y quieres descargarlos localmente antes de abrir el proyecto en Overleaf nuevamente.

Creación y manejo del archivo .gitignore

El archivo .gitignore es crucial para especificar qué archivos y directorios Git debe ignorar. Para crear y manejar un .gitignore:

  1. Crea el archivo en la raíz del proyecto.
  2. Edita el archivo añadiendo patrones de archivos o directorios a ignorar. Para proyectos LaTeX, es importante ignorar archivos compilados: 
    
    # Ignorar archivos compilados de LaTeX
    *.aux
    *.bbl
    *.blg
    *.log
    *.pdf
    *.synctex.gz
    *.fls
    *.fdb_latexmk
    
    # Ignorar archivos del sistema
    .DS_Store
    Thumbs.db
    
    # Ignorar carpetas de editor
    .vscode/
    .idea/
  3. Añade y confirma el archivo .gitignore: 
    
    git add .gitignore
    git commit -m "Añadir archivo .gitignore para proyecto LaTeX"

El uso efectivo de .gitignore ayuda a mantener tu repositorio limpio y evita el seguimiento de archivos innecesarios o sensibles.

Alternativas a GitHub

Aunque GitHub es la plataforma más popular para alojar repositorios Git, existen alternativas notables:

  • GitLab: Ofrece repositorios privados ilimitados en su plan gratuito y puede instalarse en servidores propios. Destaca por su sistema integrado de CI/CD.
  • Bitbucket: Desarrollado por Atlassian, se integra bien con Jira y otras herramientas de la suite. Ideal para equipos pequeños que necesitan repositorios privados.

Introducción a LaTeX

¿Qué es LaTeX y por qué es útil para la escritura académica?

LaTeX es un sistema de composición de textos de alta calidad, especialmente diseñado para la producción de documentos técnicos y científicos. Creado por Leslie Lamport en 1984, LaTeX es una capa de macros construida sobre el motor de composición tipográfica TeX, desarrollado por Donald Knuth.

LaTeX es particularmente útil para la escritura académica por varias razones:

  1. Calidad tipográfica superior: LaTeX produce documentos con una apariencia profesional y consistente.
  2. Gestión de referencias y bibliografías: Facilita la creación y manejo de citas y bibliografías extensas.
  3. Escritura de fórmulas matemáticas: Ofrece un sistema robusto para la composición de ecuaciones complejas.
  4. Separación de contenido y formato: Permite a los autores centrarse en el contenido, dejando el formato al sistema.
  5. Control preciso sobre la estructura del documento: Ideal para documentos largos como tesis o libros.
  6. Estabilidad y portabilidad: Los documentos LaTeX son archivos de texto plano, fáciles de versionar y compartir.

Interfaz de Overleaf - Guía Visual

Cuando abras un proyecto en Overleaf, verás la siguiente estructura:

  • Panel izquierdo (Project panel):
    • Lista de todos los archivos del proyecto.
    • Puedes crear nuevos archivos, carpetas y subir imágenes.
    • Acceso rápido a archivo main.tex (archivo principal).
  • Panel central (Editor):
    • Aquí escribes el código LaTeX.
    • Incluye numeración de líneas, colores de sintaxis y auto-indentación.
    • Busca y reemplaza con Ctrl+H (o Cmd+H en Mac).
  • Panel derecho (PDF Preview):
    • Vista previa del documento compilado en tiempo real.
    • Actualiza automáticamente conforme escribes.
    • Puedes descargar el PDF final.

Botones principales:

  • Compile (verde): Compila el documento y muestra el PDF. Se ejecuta automáticamente.
  • Share: Comparte el proyecto con otros estudiantes mediante un enlace o email.
  • History: Ver el historial de cambios (versiones anteriores del documento).
  • Menu: Acceso a opciones avanzadas como descargar proyecto, sincronización con GitHub, etc.

Limitaciones de Overleaf (versión gratuita sin licencia)

Si no accedes a través de la licencia de campus, la versión gratuita de Overleaf tiene estas limitaciones:

  1. Tiempo de compilación: Máximo 20 segundos por compilación.
  2. Colaboradores: Solo 1 colaborador por proyecto (otros pueden ver con enlace compartido).
  3. Tamaño de material editable: Máximo 7 MB por proyecto.
  4. Tamaño de archivo individual: Máximo 2 MB por archivo.
  5. Sincronización con GitHub: No disponible en versión gratuita (sí con licencia campus).

Nota: Para este curso, estas limitaciones NO son un problema. Trabajaremos con documentos razonables en tamaño.

✏️ Ejercicio 7: Editar en Local y Subirlo a GitHub (20 minutos)

  1. En tu ordenador, abre el archivo main.tex con un editor de texto (Notepad, VS Code, etc.).
  2. Haz un cambio pequeño (ej: cambia el título).
  3. En la terminal, ejecuta: 
    git status
  4. Verás que main.tex fue modificado.
  5. Haz commit de los cambios: 
    git add .
git commit -m "Actualización del título desde editor local"
  6. Sube a GitHub: 
    git push origin main
  7. Ve a tu repositorio en GitHub y verifica que el cambio está ahí.

✏️ Ejercicio 8: Clonar tu Repositorio de GitHub (10 minutos)

  1. En una carpeta nueva en tu ordenador, abre una terminal.
  2. Clona tu repositorio: 
    git clone https://github.com/tu-usuario/2025TuApellido.git
  3. Entra en el directorio: 
    cd 2025TuApellido
  4. Abre main.tex y verifica que está el contenido correcto.
